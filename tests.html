🔥 Domain delivery integration test — calls the real RPC

The end-to-end PremiumAutomation.RunDomainDeliveryE2E RPC validates the full Wix → DomainVendorService → Dynadot → three-way verification path. The sled drives the BO Fire Console UI at wix-bo.com/fire-console to invoke it.

UF-RPC · Domain delivery E2E via PremiumAutomation ✓ passing real RPC call @rpc
Navigates to BO Fire Console with the pre-built RunDomainDeliveryE2E request body (dry_run=true by default). Clicks Fire. Captures the gRPC-over-HTTP response from /fire-console/invoke/. Asserts the TldE2EResponse shape: results[] with tld, status (DRY_RUN | SUCCESS | FAILURE | SKIPPED), registrar_verified, registration_verified, sbs_verified. Override via env vars: RPC_DRY_RUN, RPC_COST_LIMIT_USD, RPC_TLD_FILTER, RPC_SITE_ID, RPC_PAYMENT_SOURCE_ID.
📄 uf-rpc-domain-delivery.spec.ts · last run: 16.7s · sample response: TLD xyz / status=DRY_RUN / domain=testdomain<rand>.xyz / expected_domain_price=15.95 EUR / total_cost_usd=0
Run locally — dry_run (safe, no money)
npm run capture-session guyme   # one-time, captures BOTH manage.wix.com + wix-bo.com auth
AUTH_STORAGE_STATE=.auth/guyme-session.json \
  npx playwright test --config playwright.local.guyme.config.ts --headed \
  sled/tests/e2e/uf-rpc-domain-delivery.spec.ts
Run locally — REAL delivery (costs money!)
AUTH_STORAGE_STATE=.auth/guyme-session.json \
  RPC_DRY_RUN=false \
  RPC_COST_LIMIT_USD=100 \
  RPC_TLD_FILTER=xyz \
  npx playwright test --config playwright.local.guyme.config.ts --headed \
  sled/tests/e2e/uf-rpc-domain-delivery.spec.ts

# Expected: status=SUCCESS, all 3 *_verified flags true, domain_name populated,
# total_cost_usd ~ catalog price, real domain registered in the designated

Discovery spec

Discovery · List domains and observed states discovery @discovery
Walks the Domain Manager for the current user (typically guyme@wix.com via stored session), captures per-domain state + visible row actions, writes findings to test-results/discovery/<timestamp>-domain-list.json. Read-only; never mutates. Excluded from the main CI suite via testIgnore in playwright.config.ts.
📄 discovery/list-domains.spec.ts · last run on guyme@wix.com: 389 domains catalogued
Run locally (requires captured session)
# 1. Capture your session (one-time, interactive — Okta SSO + MFA in the popup)
npm run capture-session guyme

# 2. Run the discovery spec as you
AUTH_STORAGE_STATE=.auth/guyme-session.json npm run discover

How to run any of these locally

Prereqs

git clone git@github.com:wix-private/Premium-QA-Sled.git
cd Premium-QA-Sled
git checkout qa/dynadot-flows-sled-phase-1
cd premium-qa-cdd/02-Domains/dynadot-flows-sled
npm install
npx playwright install chromium

Run as the QA automation user (default — uses AUTOMATION_MASTER_KEY from ~/.zshrc)

source ~/.zshrc
npm test            # full suite, headless
npm run test:smoke  # only @smoke-tagged specs, headed

Run as yourself (Okta SSO, against your own domains)

# One-time interactive: opens Chromium → you log in via Okta → script saves session
npm run capture-session guyme

# Now run any spec as you
AUTH_STORAGE_STATE=.auth/guyme-session.json npm run test:as-user
# or a specific spec:
AUTH_STORAGE_STATE=.auth/guyme-session.json \
  npx playwright test --config playwright.local.guyme.config.ts --headed \
  sled/tests/e2e/uf-05-update-whois.spec.ts

Build the dashboard from the latest run

npm run build:dashboard
open dist/index.html
